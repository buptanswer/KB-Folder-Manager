## Architecture Design

### Separation of Concerns
```
GUI Layer (gui.py)
    ‚Üì (calls)
Backend Operations (operations.py)
    ‚Üì (uses)
Core Utilities (config, validator, indexer, utils)
```

### Key Design Decisions
1. **Zero Backend Changes**: GUI wraps existing operations without modification
2. **Thread-Safe**: Operations run in separate threads, results passed via Queue
3. **Testable**: GUI tests simulate operations without actual GUI rendering
4. **Maintainable**: Clear separation between UI and business logic

## How to Use

### Quick Start - Launch GUI
```powershell
# Recommended: Run from project root
cd KB-Folder-Manager
python kb_folder_manager_gui.py
```

### Alternative: Module Import Method
```powershell
# Set PYTHONPATH first
$env:PYTHONPATH = "$PWD"  # Windows
python -m kb_folder_manager.gui

# Or on Linux/Mac
export PYTHONPATH=$(pwd)
python -m kb_folder_manager.gui
```

**Why PYTHONPATH?** The project uses relative imports and is not installed as a package. For daily use, running from the project root is simpler.

### Testing the GUI with Sample Data
```powershell
# 1. Create test data
```
