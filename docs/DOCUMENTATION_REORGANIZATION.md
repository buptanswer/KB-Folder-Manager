## 维护建议

### 更新文档时
1. **用户相关内容** → 更新 `docs/user-guide.md`
2. **开发者内容** → 更新 `docs/developer-guide.md`
3. **版本发布** → 更新 `CHANGELOG.md` 和 `docs/release-notes/`
4. **项目概览** → 更新 `README.md`

### 避免重复
- ❌ 不要在多个文件中写相同内容
- ✅ 在一个文件中详细描述
- ✅ 其他文件通过链接引用

### 保持结构
- 新增文档应放在 `docs/` 目录下
- 按用途分类（用户文档/开发文档/发布文档）
- README.md 保持简洁，仅作为入口
